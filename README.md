# 🐑 牧羊人测评站

一个用来检测 Claude Code / OpenAI 兼容 API 中转站真伪的小工具。

## 文件结构

```
muyangren-cctest/
├── index.html          # 网站首页结构
├── css/
│   └── style.css       # 视觉样式
├── js/
│   └── main.js         # 前端交互（点击检测 → 调后端）
├── server.js           # 后端服务（零依赖，仅用 Node.js 内置模块）
└── README.md           # 本说明
```

## 怎么运行（小白入门）

### 方式 A：双击直接看页面（无后端）

1. 在文件管理器里双击 `index.html`
2. 浏览器会打开页面，可以看到完整的视觉效果
3. 此时点"开始检测"会用**模拟数据**演示流程

### 方式 B：启动后端做真实检测（推荐）

1. 在 VS Code 里打开整个 `muyangren-cctest` 文件夹
2. 按 `` Ctrl + ` `` 打开终端
3. 输入命令：

   ```bash
   node server.js
   ```

4. 看到如下输出说明启动成功：

   ```
   🐑 牧羊人测评站启动成功！
      本地预览：http://localhost:3000
   ```

5. 浏览器访问 http://localhost:3000
6. 输入要测试的中转站 Base URL 和 API Key，点击"开始黑盒检测"
7. 关闭服务：在终端按 `Ctrl + C`

## 已实现的检测维度

| 维度 | 状态 | 说明 |
|------|------|------|
| LLM 指纹验证 | ✅ 已实现 | 发送 PONG 探针，看响应是否合理 |
| 非流结构完整性 | ✅ 已实现 | 检查 `id/object/created/model/choices/usage` 字段 |
| 流结构完整性 | ✅ 已实现 | 检查 SSE 事件序列与 `[DONE]` 终止符 |
| Token 用量审计 | ✅ 已实现 | 验证 `total = prompt + completion` 等基本规则 |
| 签名校验 | ✅ 已实现 | 检查 `id` / `object` / `model` 字段格式 |
| 签名真实性 | ✅ 已实现 | 连发 2 次请求，对比 id 唯一性、model 一致性、fingerprint 稳定性 |
| 多模态能力 | ✅ 已实现 | 发送 1×1 红色 PNG，看模型能否识别为 red |

## 接下来你可以怎么扩展

1. **加历史记录**：把每次检测结果保存到本地 JSON 文件
2. **加并发检测**：把 7 个维度的检测改成 `Promise.all` 并行执行，速度提升 7 倍
3. **加分享功能**：检测完成后生成一个分享链接
4. **加用户系统**：用账号登录限制使用
